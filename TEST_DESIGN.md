# テスト設計 (TEST_DESIGN.md)

## 評価項目と期待値一覧

| カテゴリ | 評価項目 | 期待値 | テスト方法 |
| :--- | :--- | :--- | :--- |
| **認証** | 初期ログイン | Googleログイン画面が開き、許可後に「同期中」になること。 | UI操作 |
| **認証** | サイレントサインイン | ログイン済みの状態でリフレッシュすると、自動で再ログインされること。 | F5リロード |
| **認証** | ログアウト | ログアウト後に「同期を開始」ボタンに戻り、再接続されないこと。 | UI操作 |
| **同期** | 初回Push | ローカルにデータがある状態で初めて同期すると、クラウドへデータが送られること。 | コンソールログ |
| **同期** | 自動保存 | タスク追加・編集時に、即座にクラウドへ更新リクエストが飛ぶこと。 | Networkタブ確認 |
| **同期** | Pull同期 | 別デバイスで更新されたデータが、リロードや同期ボタンで反映されること。 | 2デバイス同期 |
| **UI** | PCレイアウト | サイドバー底部に同期ステータスが表示されていること。 | PCブラウザ |
| **UI** | スマホレイアウト | 画面上部にモバイルヘッダーが表示され、同期ボタンが操作できること。 | スマホ/エミュレータ |
| **移行** | マイグレーション | v0（古い形式）のデータを読み込んだ際、v1へと自動変換されること。 | 擬似データの注入 |
| **安定性** | ネットワーク切断 | オフライン時にエラーでアプリが停止せず、localStorageで動作し続けること。 | 開発者ツール(Offline) |

## 検証環境
- デスクトップ: Chrome / Edge (Latest)
- モバイル: iOS Safari / Android Chrome
- サーバー: GitHub Pages (Production)
