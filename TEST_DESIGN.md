# テスト設計 (TEST_DESIGN.md) - 機能改善 v7

## 1. 評価パターンとテスト項目

| カテゴリ | 評価項目 | 確認操作 | 期待値 |
| :--- | :--- | :--- | :--- |
| **ログイン永続化** | リロード後のセッション維持 | Googleログイン後、ブラウザをリロード(F5)する | ポップアップなしで自動的に「同期済み」状態に戻ること |
| | タブを閉じた後の挙動 | タブを一度閉じ、再度同じURLを開く | （ブラウザ仕様に準ずるが）sessionStorageがクリアされ、再ログインが必要になること |
| | ログアウト時の挙動 | 「ログアウト」ボタンをクリックする | sessionStorage 内のトークンが破棄され、リロードしても再ログインが必要になること |
| **役職フィルタ** | フィルタの切り替え | カレンダー上の役職フィルタで特定の役職を選択する | アプリ全体の「自分の役職」設定を変えずに、カレンダーの表示項目のみが切り替わること |
| | 全表示モードとの併用 | 設定で「全表示」がONの場合の挙動を確認 | フィルタの選択に関わらず、すべての項目が表示されること |
| **予定ステータス** | ステータスの変更・保存 | 予定編集画面でステータスを「完了」に変更して保存する | 保存後、再度開いた際にステータスが維持されていること |
| | カレンダー上の視覚変化 | 予定を「完了」ステータスにする | カレンダーの日付セル内のドットが薄くなり、詳細カードがグレーアウト、タイトルに打消し線が入ること |
| | バッジの表示 | 予定を「完了」ステータスにする | 詳細カードに「完了」という専用バッジが表示されること |
| **データ移行** | v7 マイグレーション | 旧バージョン(v6以下)のデータが存在する状態でアプリを起動 | `migrations.ts` が走り、すべての既存予定に `status: 'todo'` が付与されていること |

## 2. テスト方法

### 手動テスト (PC / スマートフォン共通)
1. Chrome DevTools (PC) または実機 (スマホ) でページを読み込み。
2. Google認証を実行し、データが同期されることを確認。
3. リロードボタンを押し、即座に同期状態が復帰するか確認。
4. カレンダー画面で予定を作成し、ステータスを「完了」に切り替えてデザインが変化するか確認。
5. 役職フィルタを操作し、表示項目が連動して変化するか確認。

### 自動ビルド確認
- GitHub Actions またはローカルでの `npm run build` がエラーなく完了することを確認済み。
